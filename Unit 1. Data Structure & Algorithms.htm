Segment 1.4 Stack

01.    Stack এর বৈশিষ্ট্য

l    Stack হলো একটি Non-Primitive, Linear ডেটা স্ট্রাকচার

l    এটি এক ধরনের ক্রমিক তালিকা (Ordered list) যেখানে নতুন Data Item যোগ করা বা মুছে ফেলা হয় শুধুমাত্র একটি প্রান্ত থেকেই যাকে Stack এর Top বলে

l    Stack এ ডেটা আইটেম Insert করা এবং Delete করার কাজ গুলি সর্বদা উপর দিক থেকে সম্পন্ন হয়।

l    Stack LIFO - Last In First Out বা FILO First In Last Out নীতির ওপর কাজ করে। এই নীতি অনুযায়ী যে উপাদানটি Stack এ শেষে যোগ করা হবে সেটিকেই প্রথমে Delete করা যাবে।

02.    বাস্তব জীবনের Stack এর উদাহরণ

l    লাইব্রেরীতে বই এক এর উপর এক সাজানোর ক্ষেত্রে

l    প্লেট বা গ্লাস গুলি এক এর ওপর এক সাজানোর ক্ষেত্রে

03.    কম্পিউটার সিস্টেম প্রোগ্রামের স্ট্যাক এর ব্যবহার

l    প্রোগ্রামিং এ সাধারণত ফাংশন বা Recursive Call করার সময় Stack ব্যবহার করা হয়

l    Redo বা Undo অপারেশনের ক্ষেত্রে Stack ব্যবহার করা হয়

l    Web Browser এর Browsing History তে Stack ব্যবহার হয়

l    গাণিতিক এক্সপ্রেশনে অপারেটর অনুযায়ী বন্ধনী (Parenthesis) প্রদান করার জন্য Stack ব্যবহার হয়।

l    Infix থেকে Postfix বা Prefix এক্সপ্রেশন এ রূপান্তরিত করার জন্য Stack ব্যবহার করা হয়।

04.    Stack এর বিভিন্ন অপারেশন গুলির নাম লেখ

Ans.    PUSH operation, POP operation, Peek operation, is empty, is full

05.    Push Operation:

l    এই অপারেশনের মাধ্যমে Stack এর Top এ নতুন Data Item যোগ করা যায়

l    এটি একটি Insertion Operation

l    এই অপারেশন টি ততক্ষণই কার্যকরী হয় যতক্ষণ Stack টি Full না হয়ে যায়

06.    Stack Overflow কাকে বলে

Ans.    Stack এর Top থেকে Data Item কে Insert করতে করতে যদি Stack টি Full হয়ে যায়, তাহলে Stack এর মধ্যে নতুন কোনো নতুন Data Item কে যুক্ত করা যায় না, এই পরিস্থিতিকে Stack Overflow বলে।

07.    POP Operation:

l    এই অপারেশনের মাধ্যমে Stack এর Top থেকে একটি করে Data Item বাদ দেওয়া যায়

l    এটি একটি Deletion Operation

l    এই অপারেশন টি ততক্ষণই কার্যকরী হয় যতক্ষণ Stack টি Empty না হয়ে যায়

08.    Stack Underflow কাকে বলে

Ans.    Stack এর Top থেকে Data Item কে Delete করতে করতে যদি Stack টি Empty হয়ে যায়, তাহলে Stack থেকে আর কোনো Data Item কে বাদ দেওয়া যায় না, এই পরিস্থিতিকে Stack Underflow বলে।

09.    Peek Operation: Stack এর Top থেকে Data Item কে Read করার জন্য যে অপারেশন ব্যবহার করা হয় তাকে Peek অপারেশন বলে

10.    Is Empty Operation: Stack পুরোপুরি খালি আছে কিনা যাচাই করার জন্য যে অপারেশন ব্যবহার হয় তাকে Is Empty অপারেশন বলে। এর মাধ্যমে Stack Underflow পরিস্থিতি প্রতিরোধ করা যায়।

11.    Is Full Operation: Stack পুরোপুরি ভর্তি আছে কিনা জানার জন্য যে অপারেশন ব্যবহার করা হয় তাকে Is Full অপারেশন বলে। এর মাধ্যমে Stack Overflow পরিস্থিতি প্রতিরোধ করা যায়।

12.    Arithmetic Expression কাকে বলে?

Ans.    সাধারণত বিভিন্ন Arithmetic Operator এবং Operand এর সমন্বয়ে গঠিত যে অপারেশন সম্পাদনের মাধ্যমে কোনো একটি Value পাওয়া যায় তাকে Arithmetic Expression বলে।

13.    Arithmetic Expression Notation কে তিনটি উপায় প্রকাশ করা হয়

l    Infix Notation, Postfix Notation, Prefix Notation

14.    Infix Notation:

l    এটি সবচেয়ে প্রচলিত নোটেশন যেখানে Operand গুলির মাঝখানে Operator বসিয়ে কোনো এক্সপ্রেসন কে সম্পাদন করানো হয়। যেমন: A+B

l    মানুষেরা এই নোটেশন ব্যবহার করে এক্সপ্রেশন কে সম্পাদন করে

15.    Postfix Notation:

l    এই Notation এ Operand গুলির শেষে Operator গুলিকে বসিয়ে এক্সপ্রেশন কে সম্পাদন করা হয়। যেমন: AB+

l    Computer এই নোটেশন ব্যবহার করে এক্সপ্রেশন কে সম্পাদন করে

16.    Prefix Notation:

l    এই Notation এ Operand গুলির আগে Operator গুলিকে বসিয়ে এক্সপ্রেশন কে সম্পাদন করা হয়। যেমন: +AB

l    Computer এই নোটেশন ব্যবহার করে এক্সপ্রেশন কে সম্পাদন করে

l    একে Reversed Polish Notation ও বলা হয়।

Segment 1.5 Queue

01.    Queue এর বৈশিষ্ট্য

l    Queue হলো একটি Non Primitive, Linear ডেটা স্ট্রাকচার

l    এটি এক ধরনের ক্রমিক তালিকা (Ordered list) যেখানে এক প্রান্ত থেকে নতুন Data Item যোগ করা এবং অপরপ্রান্ত থেকে Data Item কে বাদ দেওয়া হয়

l    Queue তে Data Item, Insert করা একদিকে এবং Delete করা তার ঠিক অপর দিক থেকে সম্পন্ন হয়।

l    Queue FIFO - First In First Out বা LILO Last In Last Out নীতির ওপর কাজ করে। এই নীতি অনুযায়ী যে উপাদানটি Queue তে প্রথমে যোগ করা হবে সেটিকেই প্রথমে Delete করা যাবে।

02.    বাস্তব জীবনের Queue এর উদাহরণ

l    Shopping Mall, Customer Service Centre, Ticket Counter এ দাড়িয়ে থাকা মানুষ

l    Traffic Signal এ দাড়িয়ে থাকা গাড়ি

03.    কম্পিউটার সিস্টেম প্রোগ্রামের Queue এর ব্যবহার

l    কোনো কম্পিউটার সিস্টেমের প্রিন্টারে প্রিন্ট করানো, Queue এর মাধ্যমে

l    কম্পিউটার সিস্টেমের প্রসেসর একাধিক কাজকে সাজিয়ে প্রসেস করে, Queue এর মাধ্যমে

l    Client কম্পিউটার গুলি Server কম্পিউটারের কাছে Request প্রদান করে এবং উল্টোদিকে Server কম্পিউটার গুলি Client কম্পিউটারকে Response প্রদান করে Queue এর মাধ্যমে।

04.    Queue এর প্রকারভেদ

Ans.    Queue মূলত চার প্রকারের হয়

l    Normal Queue or Linear Queue

l    Circular Queue

l    Priority Queue

l    Double Ended Queue or Deque (ডেক)

05.    Queue এর REAR কী: Queue এ মোট অবস্থিত দুটি প্রান্তের, যে প্রান্ত দিয়ে Data Item Insert করা হয়, তাকে সেই Queue এর REAR বলে।

06.    Queue এর FRONT কী: Queue এ মোট অবস্থিত দুটি প্রান্তের, যে প্রান্ত দিয়ে Data Item Delete করা হয়, তাকে সেই Queue এর FRONT বলে।

07.    Queue এর বিভিন্ন অপারেশন গুলির নাম লেখ

Ans.    ENQUEUE, DEQUEUE, PEEK, IS EMPTY, IS FULL

08.    Enqueue Operation:

l    এই অপারেশনের মাধ্যমে Queue এর REAR প্রান্ত থেকে নতুন Data Item যোগ করা যায়।

l    এটি একটি Insertion Operation

l    এই অপারেশন টি ততক্ষণই কার্যকরী হয় যতক্ষণ Queue টি Full না হয়ে যায়

l    Queue Full হওয়ার পর Data Item, Insert করতে গেলে যে পরিস্থিতি তৈরি হয় তাকে, Queue Overflow বলে।

09.    Dequeue Operation:

l    এই অপারেশনের মাধ্যমে Queue এর FRONT প্রান্ত থেকে বর্তমানে অবস্থিত Data Item কে বাদ দেওয়া যায়।

l    এটি একটি Deletion Operation

l    এই অপারেশন টি ততক্ষণই কার্যকরী হয় যতক্ষণ Queue টি Empty না হয়ে যায়

l    Queue Empty হওয়ার পর Data Item, Delete করতে গেলে যে পরিস্থিতি তৈরি হয় তাকে, Queue Underflow বলে।

10.    Peek Operation: Queue এর Front থেকে Data Item কে Read করার জন্য যে অপারেশন ব্যবহার করা হয় তাকে Peek অপারেশন বলে

11.    Is Empty Operation: Queue পুরোপুরি খালি আছে কিনা যাচাই করার জন্য যে অপারেশন ব্যবহার হয় তাকে Is Empty অপারেশন বলে। এর মাধ্যমে Queue Underflow পরিস্থিতি প্রতিরোধ করা যায়।

12.    Is Full Operation: Queue পুরোপুরি ভর্তি আছে কিনা জানার জন্য যে অপারেশন ব্যবহার করা হয় তাকে Is Full অপারেশন বলে। এর মাধ্যমে Queue Overflow পরিস্থিতি প্রতিরোধ করা যায়।

13.    Normal Queue or Linear Queue: এটি সবচেয়ে সরল Queue, যেখানে একদিকে REAR প্রান্ত থাকে যা Data Insert এবং অন্যদিকে FRONT প্রান্ত থাকে যা Data Delete করার কাজ করে।

16.    Circular Queue:

l    এই Queue তে শেষ Data Element এর পরই প্রথম Data Element যুক্ত হতে পারে

l    এই Queue টি চক্রাকারে অবস্থান করে তাই, একে Circular Queue বলে।

17.    Priority Queue:

l    এই Queue তে প্রতিটি Data Element কে অগ্রাধিকার বা Priority দেওয়া থাকে।

l    এই Queue, FIFO নীতি কে অনুসরণ করে না কারণ এখানে, অগ্রাধিকার বা Priority নীতি অনুসরণ করা হয়।

l    এই Queue তে, যে Data Item এর Priority বেশি হবে, সেই Data Item কে আগে প্রোসেস (Add বা Delete) করানো হবে

l    এই Queue তে, একাধিক Data Item এর Priority সমান হলে, সেক্ষেত্রে FIFO নীতি মেনে তা Process করা হবে।

18.    Dequeue (ডেক) or Double Ended Queue:

l    Dequeue অর্থাত্‍ Double Ended Queue তে Queue এর উভয় দিকেই, REAR এবং FRONT প্রান্ত হিসাবে কাজ করে, তাই Data Insert এবং Data Delete উভয় কাজই উভয় দিকে থেকে করা যায়।

l    Dequeue দুভাগে ভাগ করা যায়:

   Input Restricted Dequeue: Data Item কে শুধুমাত্র REAR থেকে Insert কিন্তু FRONT ও REAR উভয় দিক থেকেই Delete করা যায়।

   Output Restricted Dequeue: Data Item কে শুধুমাত্র FRONT থেকে Delete কিন্তু FRONT ও REAR উভয় দিক থেকেই Insert করা যায়।

l    Dequeue, Web Browser এর Page গুলির মধ্যে Back এবং Forward করার জন্য, Operating System এর Task Schedule এর জন্য, Cache Memory Management এর কাজে ব্যবহার হয়।
